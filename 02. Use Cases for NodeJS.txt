언제 그리고 왜 Node.JS 를 사용해야 할까?

백엔드 서버를 구축할 때, python 의 장고 , PHP 의 라라벨 같은 것들이 있다고 한다. 

근데 왜 Node.JS 를 사용하는 것인가?

Javascripts를 너무 사랑해서 프론트엔드 백엔드 다 Javascripts로 만들고 싶을때 
(만약에 Javascripts를 싫어 한다면 굳이 백엔드를 Javascripts로 만들어 제끼진 않을 것이야)

Javascripts는 좋든 싫든 프런트에서 써야됨. 왜냐면 다른 프로그래밍 언어가 없거든 (물론 asp.net blazor 를 통한 C# 은 있다.)
근데 이것을 백엔드에서도 쓸수 있는 것임.

만약에 Javascripts 가 존나게 싫더라도 프런트에는 써야 함. 반면에 백엔드의 경우에는 대안이 있는것이다. (Python, Ruby, PHP, .NET)

- 또한 만약에 내가 customize를 하고 싶고, 내가 능력이 존나 좋아 그렇다면 Node.JS 야
- 기본이 있고 이것저것 필요한게 들어있는걸 원한다면 장고나 라라벨이다.

왜냐면 Node.JS 는 거의 아무것도 안들어 있음 작은 블럭들을 쌓아서 큰 성을 짓는 거라고 생각하면됨 그게 Node.JS 임. 
어떤사람들은 이걸 좋아하고 어떤사람들은 싫어함.

만약 장고를 선택한다면 장고의 경우 큰 성이 이미 갖춰져 있어서 내가 하려면 사용법을 배우고 사용하면된다. 

Django 와 node.JS 의 큰 차이다.
Node.JS : Javascripts 로 모든걸 해봐
Django : 엄청 큰 도구 사용하려면 먼저 배워야됨.

노마드햄의 경우 Node.JS 를 좋아하심 0 부터 자기가 원하는것을 만드는것을 좋아하시기 때문이다.
Django 는 기능이 100% 면 30% 밖에 안쓴다고함.

또한 

많은 데이터를 다룰때에는 Node.JS 가 낫다고 하심
Database 생성, 삭제 사용자 전송 등 nodeJS 는 데이터를 다루는 능력이 매우좋다.

근데 만약 서버의 하드웨어적 측면 (램 / 메모리 等) 에 접근해야 하는 경우라면 node.js는 무쓸모다.
이런경우가 있기도 함 데이터사이언스라던지, 고용량의 이미지 수정이라던지 
유튜브에 동영상올리는 것도 영상을 올리면 4개의 형식으로 바꾸고 오디오는 또 추출해서 저작권 있는지 확인하고 그럼
이런거는 nodeJS 가 못함, javascripts 자체가 하드코어한 하드웨어 처리를 위해 설계된게 아니기 때문이다. 
>> 이런거를 잘하는 친구가 Python Django 이다.

서버의 램이나 하드디스크를 써야하고, 바이트단위의 작업이나 비디오 인코딩 디코딩 같은 어려운작업의 경우 node.JS 를 사용하지 않는다.

단순히 data를 저장하고 생성하고 보여주고 삭제하고 이런걸 실시간으로 빠르게 하고 싶다면 nodeJS 는 fucking Best 선택지가 될것이다.
또한 채팅기능을 구현하려면 python은 좋은 선택지는 아니다. 
python 은 비동기 언어가 아니라서 실시간 처리를 할 수 있는 요소가 없다. 
채팅을 만들려면 for sure nodeJS 필요할것

인스타그램이면 django 가 맞을 것이고
uber 의 경우는 nodeJS 가 맞을 듯 왜냐면 실시간으로 위치보여주고 하니까. 

정적인 웹사이트에는 nodeJS 장고 라라벨 모두 훌륭하다.
